📝 セットアップ手順

回答に応じて、あなたに必要なステップだけ表示します

1 Claude Max に契約する ⏱️ 5分

  1. ブラウザで claude.ai を開く
  2. 「Sign Up」からアカウントを作成（会社のメールアドレスでOK）
  3. ログイン後、左下のアイコン → 「Upgrade」
  4. Max（月$100 ≒ 約15,000円）を選択
  5. クレジットカード情報を入力して契約完了
    💳 支払い方法がわからない場合は西村に聞いてください
⚠️ ProではなくMaxを選んでください。Proだと使用量の上限にすぐ到達します。

2 Obsidianをインストールする ⏱️ 5分

  1. ブラウザで obsidian.md を開く
  2. 「Get Obsidian for free」をクリック
  3. お使いのOS用をダウンロード
  4. ダウンロードしたファイルを開いてインストール

3 Vaultを作成する ⏱️ 3分

  1. Obsidianを起動
  2. 「Create new vault」（日本語表示の場合は「新しい保管庫を作成」）をクリック
  3. Vault name: MyVault（好きな名前でOK）
  4. Location: 「書類」（Documents）フォルダを選択
  5. 「Create」をクリック
💡 作成したら一旦Obsidianは閉じてOK。この後の作業はClaude Codeで行います。

4 GitHubアカウントを作成する ⏱️ 5分

  1. ブラウザで github.com を開く
  2. 「Sign up」をクリック
  3. メールアドレス（会社メールでOK）、パスワード、ユーザー名を入力
  4. メールに届いた認証コードを入力
  5. 完了！

5 Git for Windowsをインストールする ⏱️ 5分

  1. ブラウザで git-scm.com/downloads/win を開く
  2. 「Click here to download」をクリック
  3. ダウンロードしたインストーラーを実行
  4. すべてデフォルト設定のまま「Next」→「Install」
⚠️ Windowsの方のみ必要です。Macの方はGitが最初から入っています。

6 Claude Desktopアプリをインストール・起動する ⏱️ 10分

① インストール

  1. ブラウザで claude.ai/download を開く
  2. お使いのOS用のアプリをダウンロード
  3. ダウンロードしたファイルを開いてインストール

② 起動・ログイン

  1. アプリを起動
  2. Claudeアカウント（STEP 1で作成したもの）でログイン
  3. ブラウザが開いたら「Authorize」（承認）をクリック
  4. 「Success!」と表示されたらブラウザを閉じてアプリに戻る

③ Codeタブを開いてVaultを選択

  1. 上部メニューから 「Code」タブ をクリック
  2. 「Select folder」で、STEP 3で作成した Vaultフォルダ を選ぶ
    （「書類」→「MyVault」の中です。STEP 3で選んだ場所を思い出してください）
  3. 「このフォルダを信頼しますか？」→ 「はい」
💡 「Code」タブがClaude Codeです。ここから日本語で指示を出すだけで、ファイル作成・編集・Git操作すべてをAIが実行します。
⚠️ 「Code」タブが見つからない場合 → アプリ上部メニュー「Claude」→「Check for Updates」で最新版にアップデートしてください。（Windowsの場合は再ダウンロード）

6 Claude Code CLIをインストール・起動する ⏱️ 10分

Claude Codeの画面下部にある入力欄に、以下のプロンプトをペーストして送信してください。

# 第二の脳 初期セットアップ

私のObsidian Vaultを「第二の脳」として構築してください。

## 1. フォルダ構造の作成

以下のフォルダを作成してください：

```
00_Inbox/          ← とりあえず放り込む場所
07_DailyLog/       ← 日々のログ（自動蓄積）
10_Goals/          ← 目標管理
20_Context/        ← 業務の前提情報（AIの知識ベース）
30_Projects/       ← プロジェクト管理
40_Knowledge/      ← 普遍的なノウハウ
50_Literature/     ← 読書・記事メモ
60_Templates/      ← テンプレート
```

各フォルダに .gitkeep を作成して、空フォルダもGitで管理できるようにしてください。

## 2. テンプレートの作成

60_Templates/ に以下のテンプレートを作成してください：

### DailyLog.md
```markdown
---
title: "Daily Log - {{date}}"
type: daily
date: {{date}}
---

# {{date}} Daily Log

## 今日やったこと
-

## 学んだこと・気づき
-

## 明日やること
-

## メモ
-
```

### Project.md
```markdown
---
title: ""
type: project
status: planning
progress: 0
created: {{date}}
updated: {{date}}
---

# プロジェクト名

## 概要

## 目標

## タスク
- [ ]

## メモ
```

### Inbox.md
```markdown
---
title: ""
type: inbox
created: {{date}}
---

# タイトル

## 内容

## 次のアクション
```

## 3. CLAUDE.md の作成

~/.claude/CLAUDE.md に以下の内容を保存してください（ディレクトリがなければ作成）：

```markdown
# Global Rules

## 言語
- 日本語で応答する

## 基本方針
- 推測・創作を禁止。不明点は必ず質問する
- 複数の解釈が可能な場合は確認する
- セッション開始時にメモリファイルを読んでから応答
- ユーザーの訂正や好みは即座に保存

## ワークフロー: Plan → Execute
1. まず計画を立てて確認する
2. 承認されてから実行する
3. スコープが逸れたら計画に戻る

## 作業の可視化
- 各ステップの前後でフェーズを表示する
- 失敗したら正確なエラーを表示する

## コーディング原則
- 既存のアーキテクチャとパターンに従う
- 依頼されていない機能追加やリファクタリングはしない
- 秘密情報をコードにハードコードしない
```

## 4. Git初期設定とGitHub連携

以下の情報でGitを設定し、GitHubにプライベートリポジトリを作成して連携してください：

私にGitHubのユーザー名とメールアドレスを聞いてから、以下の設定で連携してください：
- リポジトリ名: MyVault
- 公開設定: private（非公開）

GitHub認証にはブラウザベースの認証（gh auth login）を使ってください。
ghコマンドがなければ先にインストールしてください。

## 5. 初回push

設定が完了したら、すべてのファイルをcommitしてGitHubにpushしてください。
コミットメッセージは「初期セットアップ完了」としてください。

## 6. 動作確認

以下を確認して結果を教えてください：
- [ ] フォルダ構造が正しく作成されている
- [ ] テンプレートが60_Templates/に存在する
- [ ] CLAUDE.mdが~/.claude/に保存されている
- [ ] GitHubにリポジトリが作成されている
- [ ] pushが成功している

よろしくお願いします！

          

          

            ⚠️ AIが途中で何度か「実行してよいですか？」と確認を求めてきます。すべて「はい」と答えてOKです。
          

          

            💡 GitHub認証時にブラウザが自動で開きます。「Authorize GitHub CLI」という画面が出たら、緑色の「Authorize」ボタンをクリックしてください。
