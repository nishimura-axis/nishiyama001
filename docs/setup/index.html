Obsidianのインストール

情報を保管するノートアプリをインストールします。

⏱️ 約5分
  1. ブラウザで https://obsidian.md にアクセス
  2. 画面中央の「Get Obsidian for free」ボタンをクリック
  3. あなたのパソコンに合わせてダウンロード
    • Macの場合：「Download for macOS」
    • Windowsの場合：「Download for Windows」
  4. ダウンロードしたファイルをダブルクリックして開く
  5. 画面の指示に従ってインストール（「続ける」「インストール」を押していくだけ）

Claude Codeのインストール

Claude Desktopアプリをインストールします。ターミナル不要。

⏱️ 約5分
  1. ブラウザで https://claude.ai/download にアクセス
  2. 画面中央の「Download」ボタンをクリック
  3. 自動でダウンロードが始まります
  4. ダウンロードしたファイルをダブルクリックして開く
  5. 画面の指示に従ってインストール
  6. 初回起動時のアカウント作成
    • 「Sign in」をクリック
    • 「Continue with Google」を選択（Googleアカウントで簡単にログインできます）
    • または、メールアドレスで新規登録

Vaultの作成

Obsidianで新しいVault（保管庫）を作成します。これがあなたの「第二の脳」の本体になります。

⏱️ 約3分
  1. Obsidianを起動
  2. 「Create new vault」（新しい保管庫を作成）をクリック
  3. 以下を入力：
    • Vault name: MyVault（または好きな名前）
    • Location: 「Browse」をクリックして、ドキュメントフォルダを選択
  4. 「Create」をクリック
  5. 作成したらObsidianは閉じてOKです（これ以降の作業はClaude Codeで行います）

Claude CodeでVaultを開く

Claude Codeを起動して、作成したVaultフォルダを開きます。

⏱️ 約2分
  1. Claude Codeを起動
  2. 「File」メニュー →「Open Folder」を選択
  3. STEP 3で作成したVaultフォルダを選択
    • 例：ドキュメント/MyVault
  4. 「Open」をクリック

GitHubアカウントの作成

クラウドにバックアップするためのGitHubアカウントを作成します。

⏱️ 約5分
  1. ブラウザで https://github.com にアクセス
  2. 画面右上の「Sign up」をクリック
  3. 以下を入力：
    • メールアドレス：会社のメールでOK
    • パスワード：8文字以上、数字と小文字を含む
    • ユーザー名：半角英数字（例：yamada-taro）
  4. 「Create account」をクリック
  5. メールに届いた認証コードを入力して完了！

アクセストークンの取得 ⚠️ 重要

AIがGitHubを操作するためのトークンを取得します。このステップだけは手動で行う必要があります。

⏱️ 約5分
  1. GitHubにログインした状態で、以下のURLにアクセス：
    https://github.com/settings/tokens/new
  2. 以下を入力：
    • Note: MyVault-ClaudeCode（何でもOK、識別用）
    • Expiration: 「No expiration」を選択（無期限）
    • Select scopes: 以下にチェック ✅
      • repo（一番上、すべての repo 権限）
  3. 画面下部の「Generate token」をクリック
  4. ⚠️ 重要：表示されたトークン（ghp_ で始まる文字列）をコピーして安全な場所に保存
    ※このトークンは一度しか表示されません。メモ帳などに一時的に貼り付けておいてください。

# 第二の脳 初期セットアップ依頼

私の「第二の脳」としてObsidian Vaultを構築してください。

## やってほしいこと

### 1. フォルダ構造の作成
以下のフォルダを作成してください：
- 00_Inbox（とりあえず放り込む場所）
- 10_Goals（目標管理）
- 20_Context（業務の前提情報）
- 30_Projects（プロジェクト管理）
- 40_Knowledge（普遍的なノウハウ）
- 50_Literature（読書・記事メモ）
- 60_Templates（テンプレート）
- 70_Meetings（MTGログ）

### 2. CLAUDE.md の作成
以下の内容で `CLAUDE.md` を作成してください：

---
# Obsidian Vault ルール（AXISホールディングス）

## 🏗️ Vault構造

MyVault/
├── 00_Inbox/          # Fleeting Note（スマホからのメモ）
├── 10_Goals/          # 目標管理（10Y→5Y→3Y→年→Q→月→週）
├── 20_Context/        # コンテキスト保管（会社の全情報、AIの前提）
├── 30_Projects/       # PJ管理（事業部ごと）
├── 40_Knowledge/      # Permanent Note（昇華されたナレッジ）
├── 50_Literature/     # 読書・記事メモ
├── 60_Templates/      # テンプレート
├── 70_Meetings/       # MTGログ（定例、レビュー会など）
└── _Dashboard.md      # 全体ホーム

## 📋 基本ルール

### 出力ルール
- 推測・創作を禁止。不明点は必ず `TODO:` を残す
- 事実と解釈を分離。解釈を書く場合は `Assumption:` と明示
- 新規ノート作成時は必ず該当テンプレートを使用
- 全ノートにYAMLフロントマターを必須

### 質問ルール
- 不明点は勝手に埋めず、必ず質問する
- 複数の解釈が可能な場合は確認する

## 🚀 会話開始フロー

### 必須ヒアリング（会話の最初に必ず実行）

新しい話題や作業を始める前に、以下を必ず確認する：

1. **何についての話か？**
   - 壁打ち / 要件定義 / 開発 / 振り返り / その他

2. **どの事業部/PJに関連するか？**
   - HD / AXIS-Ads / AXIS-PR / WEEVA / FanMarketing / Clay
   - 該当PJがあればPJ名

3. **読み込むべきコンテキストの確認**
   以下のコンテキストを読み込みますか？
   - 20_Context/{事業部}/（事業部情報）
   - 30_Projects/{事業部}/{PJ名}/（PJ情報）
   - その他：
   追加で読み込むべきものがあれば教えてください。

4. **ユーザーの確認を得てから読み込み開始**

## 💻 開発モード

### 開発作業時の必須参照

開発作業（実装、デバッグ、デプロイ等）を行う際は、以下を常に参照した状態で進める：

1. **コンテキスト**
   - 20_Context/{事業部}/ - 事業部の前提情報
   - 該当するL2/L3があれば全て読み込み

2. **要件定義**
   - 30_Projects/{PJ}/ 内の要件定義
   - または docs/requirements.md

3. **技術要件**
   - 30_Projects/{PJ}/System_Spec.md
   - または docs/technical-spec.md

### 開発中のルール

- 要件定義・技術要件を常に意識して実装
- 不明点は実装前に必ず質問
- 要件を勝手に変更しない
- デプロイ前は必ずユーザー確認（ローカルテスト優先）
- 進捗は 30_Projects/{PJ}/ に随時ログ

## 📁 フォルダ別ルール

### 00_Inbox
- 仕分け前のメモ置き場
- 週次で仕分け（PJ追加 / Knowledge化 / 捨てる）

### 10_Goals
- 会計年度は4月始まり（Q1=4-6月、Q4=1-3月）
- FY表記を使用（例：FY2025 = 2025年4月〜2026年3月）
- 上位目標との紐付けを必ず記載

### 20_Context
- 会社の全情報を保管（AIの前提知識として参照）
- 3層構造：L1事業部Overview → L2領域Overview → L3詳細Spec
- 開発時は必ずここを参照してから作業開始

### 30_Projects
- 事業部ごとにフォルダ分け
- PJ開始時は必ず目標との紐付けを確認
- システム開発PJはSystem_Specを追加

### 40_Knowledge
- PJ横断で使える普遍的なノウハウ
- 重要な意思決定はDecision_Logに記録

### 50_Literature
- 外部インプットは必ず「自分の言葉で」要約
- 「自分への適用」セクションを必ず埋める

### 70_Meetings
- 定例MTGやレビュー会の記録を保管

## 📝 YAML仕様

### 共通項目
- title: ノートタイトル
- type: ノート種別
- updated: 更新日（YYYY-MM-DD）

### フォルダ別追加項目

**30_Projects**
- bu: 事業部コード
- status: planning / in_progress / on_hold / completed
- priority: 数値（1が最優先）
- progress: 進捗率（0-100）
- next_action: 次にやること
- weekly_goal: 今週のゴール

## ⚠️ 禁止事項

- 機密情報の外部出力
- 要件の勝手な変更
- 確認なしのデプロイ作業
- ヒアリングなしでのコンテキスト読み込み

## 📋 基本ルール

### 出力ルール
- 推測・創作を禁止。不明点は必ず `TODO:` を残す
- 事実と解釈を分離。解釈を書く場合は `Assumption:` と明示
- 新規ノート作成時は必ず該当テンプレートを使用
- 全ノートにYAMLフロントマターを必須

### 質問ルール
- 不明点は勝手に埋めず、必ず質問する
- 複数の解釈が可能な場合は確認する

## 🚀 会話開始フロー

新しい話題や作業を始める前に、以下を必ず確認する：
1. 何についての話か？（壁打ち / 要件定義 / 開発 / 振り返り / その他）
2. どの事業部/PJに関連するか？
3. 読み込むべきコンテキストの確認
4. ユーザーの確認を得てから読み込み開始

❓ よくある質問

エンジニアじゃなくても使えますか？

はい、使えます。このガイドは非エンジニア向けに作成しています。コードを書く必要はありません。基本的にはAIと日本語で会話するだけで操作できます。

スマホからも使えますか？

現在、スマホから「第二の脳」と会話できる仕組みを準備中です。希望者には別途ご案内しますので、西村までお声がけください。

情報は安全ですか？

はい。データはあなたのPC内とGitHub（プライベートリポジトリ）にのみ保存されます。外部サービスにデータが保存されることはありません。※AIと会話する際、質問内容はAIプロバイダーに送信されます。

無料で使えますか？

基本機能は無料です。Claude Maxを使うとより快適になりますが、無料でも十分に活用できます。

ChatGPTと何が違うの？

大きく2点違います。①文脈の継続性：ChatGPTは毎回リセットされますが、Claude Codeはあなたの全ファイルを常に把握しています。②情報の蓄積：ChatGPTの回答は消えますが、Claude Codeでの会話結果はファイルとして永続保存されます。
