第二の脳 セットアップガイド

Obsidian × Claude Code × GitHub で実現する AI搭載ナレッジシステム

今日のゴール

Claude Codeで自分のVaultに書いて、GitHubにpushできる状態を作る。

所要時間：約30分（環境による）

🎯 はじめに：なぜ「第二の脳」が必要なのか

こんな経験ありませんか？

  • 📝 「あの情報どこにあったっけ…」 → Slack、メール、ドキュメント、手元のメモ…情報が散らばりすぎて探せない
  • 🤔 「前に同じこと考えたはずなのに…」 → 過去の思考や判断の理由が思い出せない
  • 😰 「頭の中がパンクしそう…」 → 管理すべき情報量が多すぎて処理しきれない
  • 💭 「この判断で合ってるのかな…」 → 壁打ち相手がいない、でも自分だけで考えても堂々巡り

これらはすべて「脳の容量不足」が原因です。

人間の脳は優秀ですが、現代のビジネス環境で求められる情報処理量には限界があります。
そこで必要なのが 「第二の脳」= 外付けハードディスク + AI秘書 という考え方です。

🤖 AIによる業務加速

  • 文字起こしからの情報整理
  • 要件定義の壁打ち
  • 過去の判断を踏まえた意思決定

🔗 知識の連結

  • バラバラだった情報がつながる
  • 関連性が見える化
  • 新しいアイデアや気づきが生まれる

Obsidianだけでもメモは取れますが、Claude Codeと組み合わせることで「AIが常にあなたの全情報を把握した状態」で会話できるようになります。これが圧倒的な差を生みます。

🔑 大事なこと：Obsidianには自分で書かない

Obsidianはただのメモアプリ。ここに自分で文字を打つ必要はありません。

全部Claude Codeに書かせます。

  • メモの作成 → Claude Codeに「これメモしておいて」
  • 情報の整理 → Claude Codeに「Inboxを整理して」
  • 振り返り → Claude Codeに「今週の振り返りまとめて」
  • GitHubへのバックアップ → Claude Codeに「pushして」

あなたがやるのは「日本語でAIに指示を出す」だけ。

Obsidianを開くのは「AIが書いてくれた内容を見る」時だけです。ノートアプリというより、AIが管理してくれる情報倉庫だと思ってください。

🚀 セットアップ開始

1Claude Max に契約する5分

  1. ブラウザで claude.ai を開く
  2. 「Sign Up」からアカウントを作成（会社のメールアドレスでOK）
  3. ログイン後、左下のアイコン → 「Upgrade」
  4. Max（月$100 ≒ 約15,000円）を選択
  5. クレジットカード情報を入力して契約完了
ProではなくMaxを選んでください。Proだと使用量の上限にすぐ到達します。

2Obsidianをインストールする5分

  1. ブラウザで obsidian.md を開く
  2. 「Get Obsidian for free」をクリック
  4. ダウンロードしたファイルを開いてインストール

3Vaultを作成する3分

  1. Obsidianを起動
  2. 「Create new vault」をクリック
  3. Vault name: MyVault（好きな名前でOK）
  4. Location: ドキュメントフォルダを選択
  5. 「Create」をクリック
作成したら一旦Obsidianは閉じてOK。この後の作業はClaude Codeで行います。

4GitHubアカウントを作成する5分

  1. ブラウザで github.com を開く
  2. 「Sign up」をクリック
  3. メールアドレス（会社メールでOK）、パスワード、ユーザー名を入力
  4. メールに届いた認証コードを入力
  5. 完了！

5Git for Windowsをインストールする5分

  1. ブラウザで git-scm.com/downloads/win を開く
  2. 「Click here to download」をクリック
  3. ダウンロードしたインストーラーを実行
  4. すべてデフォルト設定のまま「Next」→「Install」
Windowsの方のみ必要です。Macの方はGitが最初から入っています。

6Claude Desktopアプリをインストール・起動する10分

① インストール

  1. ブラウザで claude.ai/download を開く
  2. お使いのOS用のアプリをダウンロード
  3. ダウンロードしたファイルを開いてインストール

② 起動・ログイン

  1. アプリを起動
  2. Claudeアカウント（STEP 1で作成したもの）でログイン
  3. ブラウザが開いたら「Authorize」（承認）をクリック
  4. 「Success!」と表示されたらブラウザを閉じてアプリに戻る

③ Codeタブを開いてVaultを選択

  1. 左側メニューから 「Code」タブ をクリック
  2. プロジェクトフォルダの選択で、STEP 3で作成した Vaultフォルダ を選ぶ
  3. 「このフォルダを信頼しますか？」→ 「はい」
「Code」タブがClaude Codeです。ここから自然な日本語で指示を出すだけで、ファイル作成・編集・Git操作すべてをAIが実行します。
「Code」タブが見つからない場合は、アプリを最新版にアップデートしてください。

# 第二の脳 初期セットアップ

私のObsidian Vaultを「第二の脳」として構築してください。

## 1. フォルダ構造の作成

以下のフォルダを作成してください：

```
00_Inbox/          ← とりあえず放り込む場所
07_DailyLog/       ← 日々のログ（自動蓄積）
10_Goals/          ← 目標管理
20_Context/        ← 業務の前提情報（AIの知識ベース）
30_Projects/       ← プロジェクト管理
40_Knowledge/      ← 普遍的なノウハウ
50_Literature/     ← 読書・記事メモ
60_Templates/      ← テンプレート
```

各フォルダに .gitkeep を作成して、空フォルダもGitで管理できるようにしてください。

## 2. テンプレートの作成

60_Templates/ に以下のテンプレートを作成してください：

### DailyLog.md
```markdown

title: "Daily Log - {{date}}"

# {{date}} Daily Log

## 今日やったこと
-

## 学んだこと・気づき
-

## 明日やること
-

## メモ
-
```

### Project.md
```markdown
---
title: ""
type: project
status: planning
progress: 0
created: {{date}}
updated: {{date}}
---

# プロジェクト名

## 概要

## 目標

## タスク
- [ ]

## メモ
```

### Inbox.md
```markdown
---
title: ""
type: inbox
created: {{date}}
---

# タイトル

## 内容

## 次のアクション
```

## 3. CLAUDE.md の作成

~/.claude/CLAUDE.md に以下の内容を保存してください（ディレクトリがなければ作成）：

```markdown
# Global Rules

## 言語
- 日本語で応答する

## 基本方針
- 推測・創作を禁止。不明点は必ず質問する
- 複数の解釈が可能な場合は確認する
- セッション開始時にメモリファイルを読んでから応答
- ユーザーの訂正や好みは即座に保存

## ワークフロー: Plan → Execute
1. まず計画を立てて確認する
2. 承認されてから実行する
3. スコープが逸れたら計画に戻る

## 作業の可視化
- 各ステップの前後でフェーズを表示する
- 失敗したら正確なエラーを表示する

## コーディング原則
- 既存のアーキテクチャとパターンに従う
- 依頼されていない機能追加やリファクタリングはしない
- 秘密情報をコードにハードコードしない
```

## 4. Git初期設定とGitHub連携

以下の情報でGitを設定し、GitHubにプライベートリポジトリを作成して連携してください：

GitHub認証にはブラウザベースの認証（gh auth login）を使ってください。
