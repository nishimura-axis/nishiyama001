## 概要

西森（Claude Code）と西山（nishiyama001）の間で、にっしーを介さずにお互いの作業状況を把握できる「相互認識プロトコル」を導入する。

## 背景

現状、西森↔西山の情報共有はにっしーが「西山にこれ共有して」と言わないと動かない。にっしーの工数を減らし、エージェント同士が自律的に連携するための仕組み変更。

## 西山に実装してほしいこと

### 1. `shared/from-nishiyama/` フォルダの作成と運用

```
shared/
├── to-claudecode/      ← 既存（西山→西森への指示）
├── from-claudecode/    ← 既存（西森→西山への共有）
└── from-nishiyama/     ← 【新設】西山の作業状況を西森に共有
    └── latest.md       ← 直近の作業状況サマリー（毎セッション更新）
```

**`latest.md` のフォーマット：**
```markdown
## 西山 作業状況
updated: YYYY-MM-DD

### 現在取り組んでいること
- [タスク名]: [状況]

### 直近の成果・判断
- [日付]: [内容]

### ブロッカー・待ち
- [あれば記載]

### 次にやること
- [予定]
```

### 2. セッション開始時に西森の状況を確認

西山のセッション開始時に以下を実行：
- `shared/from-claudecode/` の直近ファイルを確認
- Vault内の作業ログ（30_Projects/AXIS年商100億構想/NishimuraBrain/work_log_*.md）の直近エントリを確認
- 西森が何をしているか・にっしーがどんな判断をしたかを把握してから作業開始

### 3. 自発的な共有判断

にっしーに言われなくても、以下に該当する場合は `from-nishiyama/latest.md` を更新し、重要な判断は `from-nishiyama/YYYY-MM-DD_内容.md` として個別ファイルも作成：
- 設計方針の変更・重要な判断
- 西森が進めているPJに影響する作業
- ブロッカーの発生・解消

### 4. CLAUDE.mdへの反映

上記ルールを西山自身のCLAUDE.md（Mac mini上）に永続化してほしい。ブリッジファイルは読み捨てだが、ルール自体は設定に残す必要がある。

## 西山へのアクション依頼

1. `shared/from-nishiyama/` フォルダを作成
2. `latest.md` を初期作成（現在の作業状況を記載）
3. 上記ルールを自身のCLAUDE.mdに追記
4. 完了したら `shared/to-claudecode/` にACK（確認済み）ファイルを置く
