# 4号タスクボード委任設計メモ（draft v0.1）

## 目的
tasks.jsonの管理を4号に委任し、各エージェントのタスク状態を4号が一元管理する。

## 現状
- tasks.jsonは各エージェント（001/002/003等）が個別に持つ
- 更新は手動 or 各エージェントが自分で書き換え
- 横断的な進捗把握はにっしーが直接確認

## 委任後のイメージ
1. **4号がtasks.jsonの更新権限を持つ**
   - 各エージェントは4号にステータス報告 → 4号が更新
   - にっしーも4号経由でタスク追加・変更

2. **4号の役割**
   - タスク状態の一元管理（CRUD）
   - 定期的な進捗サマリー生成
   - ブロッカー検知 → にっしーにアラート
   - エージェント間の依存関係トラッキング

3. **通信フロー**
   ```
   にっしー ←→ 4号（タスクボード）←→ 001/002/003/005/006
   ```

4. **必要なもの**
   - 4号がsessions_sendで各エージェントに問い合わせできる通信経路
   - tasks.jsonのスキーマ統一（現状バラバラの可能性）
   - 4号のSOUL.mdにタスク管理者としての役割定義
