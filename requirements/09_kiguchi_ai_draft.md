## 想定アーキテクチャ（仮説）

```
Plaud Note → 録音 → 文字起こし → ナレッジDB → AI応答
```

### RAGアーキテクチャ選定メモ（2026-03-04 19:30）

#### ベクトルDB比較

| DB | 特徴 | コスト | 推奨度 |
|---|---|---|---|
| Chroma | ローカル完結、Python native、セットアップ簡単 | 無料 | ★★★ PoC向き |
| Qdrant | 高性能、フィルタリング強い、Docker対応 | 無料（self-host） | ★★★ 本番向き |
| Pinecone | フルマネージド、スケール容易 | 有料（$70~/月〜） | ★★ 規模次第 |

**推奨**: PoC → Chroma、本番 → Qdrant（self-host）。木口AI程度の規模ならQdrantで十分。

#### 文字起こしパイプライン案

```
Plaud Note アプリ
  → 手動エクスポート（テキスト/PDF）
  → Google Drive or Dropbox 共有フォルダ
  → ファイル監視スクリプト（watchdog）
  → Whisper API で再文字起こし（話者分離精度向上）
  → チャンク分割 → Embedding → Qdrant
```

**なぜWhisperで再処理するか**:
- Plaud内蔵の文字起こし品質が未検証
- Whisper APIなら話者分離 + タイムスタンプが安定
- 音声ファイルがエクスポート可能ならPlaud文字起こし不要

#### チャンク戦略
- 会話単位（話者交代でセグメント分割）
- 1チャンク = 1質問＋1回答のペア
- メタデータ: 日時、話者、業務カテゴリ
- オーバーラップ: 前後1文を含める（文脈維持）
