# 分析メニュー — 日別推移・変動検知ページ設計書

> 2026-03-04 にっしー指示に基づく初版（西山作成）

## 概要

各案件×媒体の日別推移を表示し、変動している指標をAIが自動で洗い出すページ。
にっしーが普段モニター2つで手動でやっている「先月と今月の比較→改善点抽出」をAXADに搭載する。

## ユースケース

1. 運用担当者が案件を開く → 日別推移がグラフ+テーブルで表示される
2. AIが自動で「変動が大きい指標」をハイライト
3. 利益が出ていた月との差分をAIが検出 → 改善ポイントを提示
4. フィルターで絞り込み → 個人 / 課 / 案件

## 画面構成

### ヘッダー
- フィルター: 個人 / 課 / 案件（複数選択可）
- 比較対象期間: 任意選択（デフォルト: 直近半年の平均）
- 表示期間: 今月 / 先月 / 任意範囲

### メインエリア
- **日別推移グラフ**（折れ線）: imp / click / CV / CPA / ROAS / 利益
- **日別テーブル**: 同指標の数値一覧
- **比較ビュー**: 選択した比較対象期間とのオーバーレイ表示

### AI変動検知パネル（右サイドまたは下部）
- 変動が大きい指標のリスト（降順）
- 各指標について:
  - 変動幅（%）
  - 変動の方向（↑↓）
  - 比較対象期間との差分
  - AIによる推定原因（可能であれば）
  - 推奨アクション

## フィルター仕様

| フィルター | 値 | デフォルト |
|-----------|-----|-----------|
| 粒度 | 個人 / 課 / 案件 | 案件 |
| 案件 | 全案件リストから選択 | 全件 |
| 媒体 | Google / Yahoo / Meta / LINE等 | 全媒体 |
| 比較対象期間 | 任意選択 | 直近半年の平均 |
| 表示期間 | 任意 | 今月 |

## AI変動検知ロジック（案）

1. 日別データを比較対象期間の同曜日平均と比較
2. 標準偏差の1.5倍以上の変動をアラート対象
3. 複数指標が同時に変動 → 根本原因の推定（例: imp減→click減→CV減 = 配信量の問題）
4. 利益が出ていた月のパターンとの乖離度をスコア化

## 精査ページ（07_review_page.md）との関係

- 精査ページ = 個別案件の深掘り・是正アクション
- 分析ページ = 全体俯瞰・変動検知・比較分析
- 分析ページで異常を検知 → 精査ページで詳細確認、という導線
