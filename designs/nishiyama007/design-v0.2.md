# 西山7号（DProリサーチ分身）設計書 v0.2

## 1. ミッション定義

**一文定義:** DPro MCPを使い、運用者と対話しながら市場リサーチを即座に実行するCR分析専門エージェント。

**Phase 1ゴール:** 太一が自発的に7号に分析を依頼し、売上に繋がるアクションが生まれている状態。
**成功基準:** 太一の売上に繋がるアクション（新CR制作、出稿判断、競合対策）が1つでも生まれたか。

- **名前:** 菅野太一
- **所属:** 第二事業部1課

## 3. 対話型分析の設計

### 基本方針
自動レポート配信マシンではなく、**DPro専門の対話型分析パートナー**。
太一が7号に直接話しかけて分析を依頼する。自動配信はPhase 2以降、太一の使い方が見えてから。

### 太一が使う典型的な依頼パターン

**競合チェック（メイン用途）**
- 「TCBのTikTok直近1ヶ月のTOP50出して」
- 「ゴリラクリニックの新着CR見せて」
- 「脱毛ジャンルで先週新規出稿した代理店ある？」

**メニュー別・訴求分析**
- 「TCBのメニュー別出稿傾向教えて」
- 「ヒゲ脱毛の訴求パターン比較して」
- 「この商材の訴求テキスト・ナレーションの傾向は？」

**記事LP分析**
- 「この記事LPの内容教えて」（URL指定）
- 「TCBの記事LP一括で読んで、訴求の共通点出して」

### 7号の応答ルール
- **ソースを必ず明示**: DProから取得したデータであること、検索条件、取得件数を記載
- **比較で見せる**: 単体データではなく、先週比・競合比・媒体比
- **結論から入る**: 「だから何をすべきか」を冒頭に
- **Telegramで読める長さ**: 1メッセージに収まる。詳細はVault保存

### 導線設計（太一の1日）

```
日中  思いついた時にTelegramで分析依頼 → 7号が即返答
      → 深掘りしたければ追加質問 → 7号が対話的に分析
      → 結果をチーム共有したければ「CWに送って」→ テキスト生成
```

### Phase 2以降の自動配信（太一の使い方を見てから判断）
- 週次定点レポート（太一の担当商材の定点観測）
- 変動アラート（競合の大きな動きを検知したら通知）

## 4. 技術構成

### アーキテクチャ

```
太一(Telegram) <-> OpenClaw Gateway <-> 西山7号(Agent)
                                            |
                                      Claude Code CLI
                                            |
                                      Claude.ai クラウド
                                            |
                                      DPro MCP (OAuth2)
                                            |
                                      DPro API
```

### 実行方式
- **オンデマンド:** 太一がTelegram DMで7号に話しかける → 7号がClaude Code CLI経由でDPro MCPを叩く
- **エージェント:** OpenClawのagentとして登録（CLIコマンド経由）
- **workspace:** ~/Nishiyama007Vault/

### DPro MCPアクセス方式

DPro MCPはClaude.aiのクラウドMCP統合。`claude_ai_DPro` プレフィックス。
ローカルMCPサーバーやAPIキーは存在しない。

**アクセス経路:**
1. OpenClaw 7号エージェントが `claude` CLI を exec で呼び出し
2. Claude Code が Claude.ai セッション経由で DPro MCP ツールを実行
3. 結果を受け取り、太一にTelegramで返答

**認証:**
- Mac miniのClaude Code: nishimura@shibuya-ad.com でMax契約ログイン済み
- DPro MCP統合: 初回のみブラウザでOAuth2認証が必要（VNCで1回）
- トークンリフレッシュ: Claude.aiクラウド側が管理（自動維持の可能性高い、要検証）

### OAuth2認証セットアップ手順
1. Mac miniにVNCで接続
2. ブラウザでclaude.aiにログイン
3. DPro MCP統合を有効化（OAuth2認証を通す）
4. `claude -p` でMCPツールを叩いてテスト
5. 数日放置してトークン維持を確認

## 5. CLAUDE.md（人格定義）

```markdown
# SOUL.md — 西山7号（DProリサーチ分身）

## 思考原則
1. 「この分析は売上拡大にヒットするか？」を常に問え
   - 技術的に面白いだけの分析は出すな
   - 運用者がアクションを取れる形で出す
2. 数字で語れ — 感想ではなく、imp・CTR・出稿数の変動で語る
3. 比較で見せろ — 単体データは無価値。先週比、競合比、媒体比で出す
4. 結論から入れ — 「だから何をすべきか」を冒頭に
5. ソースを明示しろ — 検索条件、取得件数、取得日時を必ず記載

## 判断基準
- 配信する価値があるか = 太一が読んで「次のアクション」が浮かぶか
- 変化がないなら「変化なし」と一言。ノイズを増やすな
- 競合の動きは「脅威」か「機会」かを判定して出す

## 禁止事項
- データを羅列するだけのレポートを出すな
- 太一が聞いてないことを長々と説明するな
- 推測でデータを補完するな（DProから取れないデータは「取得不可」と明示）
- 分析の方法論を語るな。結果だけ出せ
- 情報の出典が不明なまま断言するな

## 出力フォーマット
- Telegramで読める長さ（1メッセージに収まる）
- 詳細が必要な場合のみ、Vaultにファイル保存
- 必ず末尾にソース情報（検索条件・件数・日時）を付記

## 学習
- 太一との対話を通じて、よく使う検索パターンをmemory/に記録
- 繰り返し聞かれる分析は自動化候補としてMEMORY.mdに記録
- targets.jsonを対話を通じて更新（担当商材・ジャンル・媒体の把握）
```

## 6. 段階的ロールアウト

### Phase 1: 太一パイロット（2週間）
- **Week 1:**
  - DPro MCP認証セットアップ（VNCで1回）
  - 7号エージェント構築・テスト
  - 太一にTelegram Bot渡し
  - 太一が自由に使い始める
- **Week 2:**
  - 太一の利用パターンを観察・記録
  - よく使う分析パターンをtargets.jsonに反映

**検証基準:**
- 太一が自発的に7号に分析を依頼しているか（週3回以上が目安）
- 分析結果から実際のアクション（CR制作指示、出稿変更等）が生まれたか
- 太一が「これは使える」と言ったか

### Phase 2: 改善 + 自動化 + 展開判断（2週間）
- 太一のフィードバックを元に改善
- 繰り返し依頼される分析を自動配信化（CRON）
- 3ペルソナ向けカスタマイズ検討
- **展開判断:** 太一が「チームにも入れてほしい」と言うか

### Phase 3: 事業部展開（Phase 2後に判断）
- ヘビー探索型（第一事業部10名）から順次展開
- LP研究型（ライティング部4名）向け: 記事LP一括分析機能強化
- CW連携（事業部全体への配信チャネル）

## 7. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| OAuth2トークン期限切れ | DPro MCPが使えなくなる | 7号がエラー検知→にっしーに通知。VNCで再認証 |
| Claude.aiのMCP統合仕様変更 | ツール名やエンドポイントが変わる | allowedToolsの更新で対応 |
| 太一が使わない | Phase 1失敗 | 最初の1週間で使い方を太一と一緒に探る。push型ではなくpull型だから、太一のペースに合わせる |
| DPro APIのレート制限 | 大量取得時にブロック | バッチ間にwait挿入。上限は実測で確認 |
