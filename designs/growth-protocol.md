# 🧬 自律成長プロトコル

使えば使うほど賢くなる仕組み。全AIエージェントに共通適用するルール。

## Memory Structure（成長に必要なファイル）

```
memory/
├── YYYY-MM-DD.md         ← 日次ログ（従来通り）
├── patterns.md           ← 蒸留された知見（抽象化されたパターン）
├── user_profile.md       ← 対話相手の嗜好・思考パターン
├── topics.md             ← 相手が繰り返す関心テーマ（先回り用）
```

## 4つのルール

### 1. 記録（毎対話）
- 対話ごとに `memory/YYYY-MM-DD.md` に要約を追記（従来通り）
- 相手が深掘りしたテーマ、聞き直した箇所、使わなかった情報を記録

### 2. 昇華（5対話ごと）
- 5対話ごとに直近の対話を振り返る
- 個別の事実ではなく**パターン**として `memory/patterns.md` に蓄積
- ❌「3/12の分析は好評だった」（事実の記録）
- ✅「TikTok案件では媒体CVと実CVの乖離を先に出すと判断が速くなる」（パターン）

振り返りで出す問い：
- 相手が深掘りした箇所 = 関心が高いテーマ → `user_profile.md` 更新
- 相手が聞き直した箇所 = 説明が不十分 → 出力フォーマット改善
- 相手が使わなかった情報 = ノイズ → 次回から省略

### 3. 先回り（受け身→能動）
- `memory/topics.md` で相手が3回以上聞いたテーマを検知
- 次の対話の冒頭で、聞かれる前にそのテーマの最新データを先出し
- 「便利な検索エンジン」から「先回りできるアナリスト」への進化

### 4. 文脈注入（claude -p呼び出し時）
- `claude -p` に渡すプロンプトには **必ず** 以下を含める：
  - `memory/patterns.md` から関連パターン
  - `memory/user_profile.md` から相手の嗜好
  - 過去の関連分析結果
- 個別の対話ログではなく、蒸留された知見を注入する

## CLAUDE.md / AGENTS.md に追記するセクション

```markdown
## 🧬 Growth Protocol

### Memory Files
- `memory/patterns.md` — 蒸留された知見。個別事実ではなくパターンで記録
- `memory/user_profile.md` — 対話相手の嗜好・思考パターン
- `memory/topics.md` — 繰り返し出るテーマ（3回以上→先回り対象）

### Rules
1. **Record** — 毎対話後にmemory/YYYY-MM-DD.mdに要約追記
2. **Distill** — 5対話ごとにpatterns.mdにパターン化。事実→抽象化
3. **Profile** — 振り返り時にuser_profile.mdを更新
4. **Anticipate** — topics.mdで3回以上出たテーマは次回冒頭で先出し

### Distillation Questions（振り返り時に自問）
- 相手が深掘りした箇所は？ → 関心テーマ → topics.md更新
- 相手が聞き直した箇所は？ → 説明改善 → patterns.md更新
- 相手が使わなかった情報は？ → ノイズ → 次回省略

### Context Injection
外部ツール（claude -p等）に渡すプロンプトには必ず：
- patterns.mdから関連パターン
- user_profile.mdから相手の嗜好
を含める。生ログではなく蒸留された知見を注入する。
```

## リサーチ出力ルール（7号等の分析エージェント向け）

分析・リサーチの出力には **必ず** 以下を含める：
- **動画URL**（YouTube/TikTok/Instagram等の実際の動画リンク）
- **記事URL**（遷移先LP/記事のリンク）
- **ソース情報**（検索条件・件数・日時）

ソースなしの分析は分析じゃない。相手が「根拠は？」と聞く手間をゼロにする。

## フィードバック閉ループ

- 毎回末尾に「この分析は役に立った？」を付ける
- 相手の反応を `memory/YYYY-MM-DD.md` に記録
- 振り返り時に「なぜ刺さったか/ズレたか」を分析 → `patterns.md` に追加
- 同じミスを繰り返さない。同じパターンで3回ズレたらCLAUDE.md自体を修正
